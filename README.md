# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов. Реализует интерфейс `IApi`.

```typescript
export interface IApi {
get<T extends object>(uri: string): Promise<T>;
post<T extends object>(uri: string, data: object, method?: ApiPostMethods): Promise<T>;
}
```
Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

Раздел описывает интерфейсы и типы данных, используемых в приложении.

#### Тип `TPayment`

Тип описывает способы оплаты товара.

```typescript
export type TPayment = 'online' | 'cash';
```

**Описание значений:**
* `'online'` — онлайн‑оплата (картой или электронными деньгами);
* `'cash'` — оплата наличными при получении.

#### Интерфейс `ICard`

Описывает структуру данных товара.

```typescript
export interface ICard {
  id: string;
  title: string;
  image: string;
  category: string;
  price: number | null;
  description: string;
}
```

**Описание свойств:**

* **`id`** — уникальный идентификатор товара (тип `string`).
* **`title`** — название товара (тип `string`).
* **`description`** — описание товара (тип `string`).
* **`image`** — ссылка на изображение товара (тип `string`).
* **`category`** — категория товара (тип `string`).
* **`price`** — цена товара (тип `number | null`). Может быть `null`, если цена не указана.

#### Интерфейс `IBuyer`

Описывает свойства объекта с данными покупателя.

```typescript
interface IBuyer {
  payment: TPayment | null;
  address: string;
  email: string;
  phone: string;
}
```

**Описание свойств:**

* **`payment`** — способ оплаты, выбранный покупателем (тип `TPayment | null`). Может быть `null` на этапе заполнения формы до выбора способа оплаты.
* **`email`** — электронная почта покупателя (тип `string`).
* **`phone`** — номер телефона покупателя (тип `string`).
* **`address`** — адрес доставки товара (тип `string`).

#### Интерфейс `IApiOrderResponse`

Описывает структуру ответа сервера на успешную отправку данных заказа.

```typescript
interface IApiOrderResponse {
  id: string;
  total: number;
}
```

**Описание свойств:**

* **`id`** — уникальный идентификатор созданного заказа на сервере (тип `string`). Присваивается сервером после успешной обработки запроса на создание заказа.
* **`total`** — итоговая сумма заказа (тип `number`), подтверждённая сервером. Значение должно совпадать с полем `total` из отправленного объекта `IOrder`.

#### Интерфейс `IOrder`

Описывает структуру данных заказа, включая информацию о покупателе (наследуется от `IBuyer`), общую стоимость и список товаров.

```typescript
interface IOrder extends IBuyer {
  total: number;
  items: string[];
}
```

**Описание свойств:**

* **`payment`** — способ оплаты, выбранный покупателем (унаследовано от `IBuyer`, тип `TPayment | null`).
* **`email`** — электронная почта покупателя (унаследовано от `IBuyer`, тип `string`).
* **`phone`** — номер телефона покупателя (унаследовано от `IBuyer`, тип `string`).
* **`address`** — адрес доставки товара (унаследовано от `IBuyer`, тип `string`).
* **`total`** — общая стоимость заказа (тип `number`). Рассчитывается на клиенте как сумма цен всех товаров в заказе. Передаётся на сервер и подтверждается им.
* **`items`** — массив идентификаторов товаров, входящих в заказ (тип `string[]`). Содержит `id` товаров из интерфейса `ICard`, которые пользователь добавил в корзину.

#### Тип `ApiPostMethods`

Определяет допустимые HTTP‑методы для выполнения запросов через API.

```typescript
type ApiPostMethods = 'POST' | 'PUT' | 'DELETE';
```

**Описание значений:**

* `'POST'` — метод для создания новых ресурсов.
* `'PUT'` — метод для полного обновления существующих ресурсов.
* `'DELETE'` — метод для удаления ресурсов с сервера.

#### Интерфейс `IApi`

Описывает интерфейс HTTP‑клиента для взаимодействия с API.

```typescript
export interface IApi {
  get<T extends object>(uri: string): Promise<T>;
  post<T extends object>(uri: string, data: object, method?: ApiPostMethods): Promise<T>;
}
```

**Описание методов:**

* **`get<T>`** — выполняет HTTP‑запрос типа GET для получения данных с указанного URI.
  * **Параметры:**
    * `uri` — строка с адресом ресурса на сервере.
  * **Возвращает:** `Promise<T>`, где `T` — обобщённый тип ожидаемого ответа.
* **`post<T>`** — выполняет HTTP‑запрос (POST, PUT или DELETE) для отправки данных на сервер.
  * **Параметры:**
    * `uri` — строка с адресом ресурса на сервере;
    * `data` — объект с данными для отправки (например, объект `IOrder`);
    * `method` (опционально) — HTTP‑метод из типа `ApiPostMethods`. По умолчанию — `'POST'`.
  * **Возвращает:** `Promise<T>`, где `T` — обобщённый тип ожидаемого ответа (например, `IApiOrderResponse`).

### Модели данных
В этом разделе будут описаны классы моделей данных.

#### Класс `CatalogData`

Класс реализует интерфейс `ICatalogData` и содержит свойства и методы для работы с массивом товаров. Управляет данными каталога: сохранением карточек товаров, получением карточек из хранилища, поиском по ID и режимом предпросмотра выбранной карточки.

```typescript
export interface ICatalogData {
    setCards(cards: ApiListResponse<ICard>): void;
    setPreviewCard(card: ICard): void;
    getCards(): ICard[];
    getPreviewCard(): ICard | null;
    getCard(id: string): ICard | undefined;
}
```

##### Свойства

* **`events`** — объект для работы с событиями (тип `IEvents`). Используется для оповещения других компонентов системы о произошедших действиях (например, о сохранении карточек товаров). Передаётся в конструктор и сохраняется в свойстве класса.
* **`cards`** — массив карточек товаров (тип `ICard[]`). Хранит все доступные карточки товаров, загруженные через метод `setCards`. Инициализируется пустым массивом.
* **`preview`** — содержит данные о карточке, выбранной пользователем для предпросмотра (тип `ICard | null`). Изначально имеет значение `null`. Устанавливается через метод `setPreviewCard`.
* **`cdn`** — базовый URL для формирования полных ссылок на изображения товаров (тип `string`). Добавляется к пути изображения из данных карточки при загрузке массива товаров. Передаётся в конструкторе.

---

##### Методы

* **`constructor(events: IEvents, cdn: string)`** — инициализирует экземпляр класса.
  * **Параметры:**
    * `events` — объект для работы с событиями (тип `IEvents`), сохраняется в свойство `events`;
    * `cdn` — базовый URL CDN‑сервера (тип `string`), сохраняется в свойство `cdn`, используется для формирования полных путей к изображениям товаров.

* **`setCards(cards: ApiListResponse<ICard>)`** — метод для загрузки и сохранения массива карточек товаров.
  * **Параметр:** `cards` (тип `ApiListResponse<ICard>`) — ответ сервера, содержащий массив товаров.
  * **Логика работы:**
    1. Извлекает массив товаров из поля `items` объекта `cards`.
    2. Для каждой карточки формирует полный URL изображения, объединяя `this.cdn` и путь из поля `image`.
    3. Сохраняет обработанный массив в свойство `this.cards`.
    4. Генерирует событие `AppEvents.CardsSaved` через `this.events.emit()`, уведомляя другие компоненты о загрузке товаров.

* **`setPreviewCard(card: ICard)`** — метод для установки карточки товара в режим предпросмотра.
  * **Параметр:** `card` (тип `ICard`) — карточка товара, которую нужно показать в предпросмотре.
  * **Логика работы:**
    1. Сохраняет переданный объект `card` в свойство `this.preview`.
    2. Генерирует событие `AppEvents.CardSaved` через `this.events.emit()`, уведомляя систему о выборе карточки.

* **`getCards()`** — метод для получения массива всех карточек товаров.
  * **Возвращаемое значение:** `ICard[]` — текущий массив карточек товаров (`this.cards`).

* **`getPreviewCard()`** — метод для получения карточки товара, установленной в режиме предпросмотра.
  * **Возвращаемое значение:** `ICard | null` — текущая карточка предпросмотра либо `null`, если ни одна карточка не выбрана.

* **`getCard(id: string)`** — метод для поиска и получения карточки товара по её уникальному идентификатору.
  * **Параметр:** `id` (тип `string`) — уникальный идентификатор искомой карточки товара.
  * **Возвращаемое значение:** `ICard | undefined` — найденная карточка товара либо `undefined`, если карточка с указанным `id` не найдена в массиве `this.cards`.
  * **Реализация:** использует метод `Array.find()` для поиска карточки по условию `card.id === id`.

##### Особенности реализации

1. **Обработка изображений:** при загрузке карточек через `setCards` путь к изображению дополняется базовым URL из `this.cdn`. Это позволяет корректно отображать картинки независимо от расположения CDN.

#### Класс `BuyerData`

Класс реализует интерфейс `IBuyerData` и содержит данные о покупателе и методы для работы с ними. Управляет хранением, валидацией и обновлением информации о покупателе, а также взаимодействует с системой событий для оповещения других компонентов.

```typescript
export interface IBuyerData {
    getUserData(): IBuyer;
    setPayment(method: TPayment): void;
    setAddress(address: string): void;
    setEmail(email: string): void;
    setPhone(phone: string): void;
    clearData(): void;
    isFieldValid(data: TPayment | null | string): boolean;
    validateUserData(): Record<string, string> | {};
    isOrderDataValid(): boolean;
    isContactsDataValid(): boolean;
    getErrors(): Record<string, string> | {};
}
```

##### Свойства

* **`events`** — объект для работы с событиями (тип `IEvents`). Используется для оповещения других компонентов системы о произошедших действиях (например, о сохранении данных пользователя). Передаётся в конструкторе и сохраняется в свойстве класса.
* **`payment`** — способ оплаты, выбранный пользователем (тип `TPayment | null`). Изначально имеет значение `null`. Устанавливается через метод `setPayment`.
* **`address`** — адрес пользователя (тип `string`). Изначально пустая строка. Устанавливается через метод `setAddress`.
* **`email`** — электронная почта пользователя (тип `string`). Изначально пустая строка. Устанавливается через метод `setEmail`.
* **`phone`** — номер телефона пользователя (тип `string`). Изначально пустая строка. Устанавливается через метод `setPhone`.
* **`errors`** — объект с сообщениями об ошибках (тип `Record<string, string>`). Хранит ошибки валидации для отдельных полей. Инициализируется пустым объектом. Заполняется методом `validateUserData()`.

##### Методы

* **`constructor(events: IEvents)`** — инициализирует экземпляр класса.
  * **Параметр:** `events` (тип `IEvents`) — объект для работы с событиями. Сохраняется в свойство `events`.
  * **Инициализация свойств:** устанавливает начальные значения для `payment` (`null`), `address`, `email`, `phone` (пустые строки), `errors` (пустой объект).

* **`getUserData()`** — возвращает полные данные пользователя в формате `IBuyer`.
  * **Возвращаемое значение:** `IBuyer` — объект с текущими значениями `payment`, `address`, `email` и `phone`.

* **`setPayment(method: TPayment)`** — устанавливает способ оплаты.
  * **Параметр:** `method` (тип `TPayment`) — выбранный способ оплаты.
  * **Логика работы:**
    1. Сохраняет значение в свойство `this.payment`.
    2. Вызывает `validateUserData()` для обновления состояния ошибок.
    3. Генерирует событие `AppEvents.PaymentSaved` с объектом `{ payment: this.payment }`.

* **`setAddress(address: string)`** — устанавливает адрес пользователя.
  * **Параметр:** `address` (тип `string`) — адрес для сохранения.
  * **Логика работы:**
    1. Сохраняет значение в свойство `this.address`.
    2. Вызывает `validateUserData()` для обновления состояния ошибок.
    3. Генерирует событие `AppEvents.AddressSaved`.

* **`setEmail(email: string)`** — устанавливает электронную почту пользователя.
  * **Параметр:** `email` (тип `string`) — email для сохранения.
  * **Логика работы:**
    1. Сохраняет значение в свойство `this.email`.
    2. Вызывает `validateUserData()` для обновления состояния ошибок.
    3. Генерирует событие `AppEvents.EmailSaved`.

* **`setPhone(phone: string)`** — устанавливает номер телефона пользователя.
  * **Параметр:** `phone` (тип `string`) — номер телефона для сохранения.
  * **Логика работы:**
    1. Сохраняет значение в свойство `this.phone`.
    2. Вызывает `validateUserData()` для обновления состояния ошибок.
    3. Генерирует событие `AppEvents.PhoneSaved`.

* **`clearData()`** — очищает все сохранённые данные пользователя.
  * **Логика работы:**
    1. Устанавливает `null` для `this.payment`.
    2. Устанавливает пустые строки для `this.address`, `this.email`, `this.phone`.
    3. Вызывает `validateUserData()` для сброса ошибок валидации.

* **`isFieldValid(data: TPayment | null | string)`** — проверяет валидность отдельного поля.
  * **Параметр:** `data` (тип `TPayment | null | string`) — значение поля для проверки.
  * **Логика:** возвращает `true`, если значение существует (не `null`, не пустая строка), иначе `false`.
  * **Возвращаемое значение:** `boolean`.

* **`validateUserData()`** — проверяет валидность всех полей данных пользователя и формирует объект с ошибками.
  * **Логика работы:**
    1. Очищает объект `this.errors`.
    2. Проверяет каждое поле с помощью `isFieldValid()`.
    3. Если поле невалидно, добавляет в `this.errors` сообщение об ошибке:
      * `payment`: «Необходимо выбрать вид оплаты»;
      * `address`: «Необходимо указать адрес доставки»;
      * `email`: «Необходимо указать email»;
      * `phone`: «Необходимо указать номер телефона».
  * **Возвращаемое значение:** `Record<string, string> | {}` — объект с ошибками или пустой объект, если ошибок нет.

* **`isOrderDataValid()`** — проверяет, достаточно ли данных для оформления заказа.
  * **Логика работы:**
    1. Вызывает `validateUserData()` для актуализации ошибок.
    2. Возвращает `true`, если отсутствуют ошибки для полей `payment` и `address` (`!this.errors.payment && !this.errors.address`).
  * **Возвращаемое значение:** `boolean`.

  * **Логика работы:**
    1. Вызывает `validateUserData()` для актуализации ошибок.
    2. Возвращает `true`, если отсутствуют ошибки для полей `email` и `phone` (`!this.errors.email && !this.errors.phone`).
  * **Возвращаемое значение:** `boolean`.

* **`getErrors()`** — возвращает объект с текущими ошибками валидации.
  * **Возвращаемое значение:** `Record<string, string> | {}` — объект `this.errors` с сообщениями об ошибках или пустой объект, если ошибок нет.

#### Класс `BasketData`

Класс реализует интерфейс `IBasketData` и содержит данные о товарах, добавленных в корзину, и методы для работы с ними. Управляет коллекцией товаров в корзине: добавлением, удалением, подсчётом общей стоимости и проверкой состояния корзины. Взаимодействует с системой событий для оповещения других компонентов об изменениях.

```typescript
export interface IBasketData {
    addCard(card: ICard): void;
    removeCard(id: string): void;
    getTotalPrice(): number;
    getCards(): ICard[];
    isInBasket(id: string): boolean;
    getCardsCount(): number;
    cleanBasket(): void;
}
```

##### Свойства

* **`events`** — объект для работы с событиями (тип `IEvents`). Используется для оповещения других компонентов системы об изменениях в корзине (например, о добавлении или удалении товара). Передаётся в конструкторе и сохраняется в свойстве класса.
* **`cardsInBasket`** — массив товаров в корзине (тип `ICard[]`). Хранит все карточки товаров, добавленные пользователем в корзину. Инициализируется пустым массивом (`[]`) при создании экземпляра класса.

---

##### Методы

* **`constructor(events: IEvents)`** — инициализирует экземпляр класса.
  * **Параметр:** `events` (тип `IEvents`) — объект для работы с событиями. Сохраняется в свойство `events`.
  * **Инициализация свойств:** устанавливает `this.cardsInBasket = []`.

* **`addCard(card: ICard)`** — добавляет товар в корзину, если его ещё нет в корзине.
  * **Параметр:** `card` (тип `ICard`) — карточка товара, которую нужно добавить в корзину.
  * **Логика работы:**
    1. Проверяет наличие товара в корзине с помощью `isInBasket(card.id)`.
    2. Если товара нет, добавляет его в массив `this.cardsInBasket` через `push()`.
    3. Генерирует событие `AppEvents.BasketChanged` через `this.events.emit()`, уведомляя систему об изменении корзины.
    4. Если товар уже есть в корзине, выводит в консоль сообщение `${card.title} уже есть в корзине`.

* **`removeCard(id: string)`** — удаляет товар из корзины по его идентификатору.
  * **Параметр:** `id` (тип `string`) — уникальный идентификатор товара, который нужно удалить.
  * **Логика работы:**
    1. Фильтрует массив `this.cardsInBasket`, исключая элемент с `item.id === id`.
    2. Обновляет `this.cardsInBasket` отфильтрованным массивом.
    3. Генерирует событие `AppEvents.BasketChanged`, уведомляя систему об удалении товара.

* **`getTotalPrice()`** — рассчитывает общую стоимость всех товаров в корзине.
  * **Логика:** использует метод `Array.reduce()` для суммирования цен всех товаров. Для каждого товара берётся `item.price`, если оно `null` или `undefined`, используется `0`.
  * **Возвращаемое значение:** `number` — сумма цен всех товаров в массиве `cardsInBasket`.

* **`getCards()`** — возвращает массив всех товаров, находящихся в корзине.
  * **Возвращаемое значение:** `ICard[]` — текущий массив товаров `cardsInBasket`.

* **`isInBasket(id: string)`** — проверяет, находится ли товар с указанным идентификатором в корзине.
  * **Параметр:** `id` (тип `string`) — идентификатор товара для проверки.
  * **Логика:** использует метод `Array.some()` для поиска товара с `item.id === id`.
  * **Возвращаемое значение:** `boolean` — `true`, если товар с таким `id` есть в корзине, `false` — если нет.

* **`getCardsCount()`** — возвращает количество товаров в корзине.
  * **Логика:** возвращает длину массива `this.cardsInBasket.length`.
  * **Возвращаемое значение:** `number` — число элементов в массиве `cardsInBasket` (общее количество карточек товаров).

* **`isEmptyBasket()`** — проверяет, пуста ли корзина.
  * **Логика:** проверяет условие `this.cardsInBasket.length === 0`.
  * **Возвращаемое значение:** `boolean` — `true`, если корзина пуста, `false` — если в корзине есть товары.

* **`cleanBasket()`** — очищает корзину от всех товаров.
  * **Логика работы:**
    1. Устанавливает `this.cardsInBasket = []`, очищая массив товаров.
    2. Генерирует событие `AppEvents.BasketChanged`, уведомляя систему о том, что корзина очищена.

---

##### Особенности реализации

1. **Защита от дубликатов:** метод `addCard` проверяет наличие товара в корзине перед добавлением. Это предотвращает дублирование карточек с одинаковым `id`.
2. **События:** класс активно использует систему событий (`IEvents`) для интеграции с другими компонентами. Событие `AppEvents.BasketChanged` генерируется при любом изменении корзины:
  * добавлении товара (`addCard`);
  * удалении товара (`removeCard`);
  * очистке корзины (`cleanBasket`).
3. **Обработка `null` цен:** в методе `getTotalPrice` используется оператор `?? 0`, чтобы корректно обрабатывать случаи, когда `price` равен `null` или `undefined`. Это гарантирует, что расчёт общей суммы не прервётся из‑за отсутствующих цен.
4. **Инкапсуляция:** свойства класса объявлены как `protected`, что ограничивает прямой доступ извне и поощряет использование публичных методов интерфейса.
5. **Эффективность операций:**
  * удаление товара реализовано через `filter()`, что создаёт новый массив без указанного элемента;
  * проверка наличия товара использует `some()`, который останавливается при первом совпадении, оптимизируя поиск.
6. **Типизация:** строго типизированные свойства и методы обеспечивают безопасность типов и упрощают поддержку кода.

  * Открывает модальное окно.
  * Добавляет класс `modal_active` к `modalElement`, делая окно видимым.
  * Подписывается на событие клавиатуры (`keyup`) для отслеживания нажатия клавиши Escape (вызывает `handleEsc`).
  * Отправляет событие `AppEvents.ModalOpen` через систему событий `events` для уведомления других компонентов о том, что модальное окно открыто.

3. `closeModal(): void`
  * Закрывает модальное окно.
  * Удаляет класс `modal_active` из `modalElement`, скрывая окно.
  * Отписывается от события клавиатуры (`keyup`), убирая обработчик `handleEsc`.
  * Отправляет событие `AppEvents.ModalClose` через систему событий `events` для уведомления других компонентов о закрытии окна.

4. `handleEsc(evt: KeyboardEvent): void`
  * Обрабатывает нажатие клавиши Escape для закрытия модального окна.
  * Принимает объект события клавиатуры `evt: KeyboardEvent`.
  * Проверяет, была ли нажата клавиша Escape (`evt.key === "Escape"`).
  * Если условие выполняется, вызывает метод `closeModal()` для закрытия окна.

**Наследуемые свойства и методы:**

Класс `ModalView` наследует от `Component` следующие ключевые элементы:
* `container: HTMLElement` — корневой DOM‑элемент компонента (в данном случае совпадает с `modalElement`);
* `render(data?: Partial<IModalView>): HTMLElement` — главный метод для отрисовки компонента. Может быть переопределён в дочерних классах для реализации специфичной логики отображения;
* `setImage(element: HTMLImageElement, src: string, alt?: string): void` — утилитный метод для работы с изображениями (может пригодиться, если внутри модального окна появится изображение).

Эти унаследованные элементы обеспечивают базовую функциональность компонента и могут быть использованы при расширении класса или настройке отображения модальных окон.
