# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
Раздел описывает интерфейсы данных, используемых в приложении.

#### Тип `TPayment`
Тип описывает способы оплаты товара.

```typescript
export type TPayment = 'card' | 'cash' | '';
```

#### Интерфейс `IProduct`

Описывает структуру данных товара.
```typescript
export interface ICard {
    id: string;
    title: string;
    image: string;
    category: string;
    price: number | null;
    description: string;
}
```

##### Описание свойств

* **`id`** - уникальный идентификатор товара.

* **`title`** - название товара.

* **`description`** - описание товара.

* **`image`** - ссылка на изображение товара.

* **`category`** - категория товара.

* **`price`** - цена товара.

#### Интерфейс `IBuyer`
Описывает свойства объекта покупателя.

```typescript
interface IBuyer {
    payment: TPayment;
    email: string;
    phone: string;
    address: string;
}
```

##### Описание свойств

* **`payment`** — способ оплаты, выбранный покупателем.
* **`email`** — электронная почта.
* **`phone`** — номер телефона.
* **`address`** — адрес доставки товара.

#### Класс `CatalogData`
Класс реализует интерфейс `ICatalogData` и содержит свойства и методы для работы с массивом товаров.

```typescript
export interface ICatalogData {
    setCards (cards: ICard[]): void;
    setPreviewCard (card: ICard): void;
    getCards(): ICard[];
}
```

##### Свойства

* **`events`** — объект для работы с событиями (тип `IEvents`), используется для оповещения других компонентов системы о произошедших действиях (например, о сохранении карточек товаров).

* **`cards`** — массив карточек товаров (тип `ICard[]`), хранит все доступные карточки товаров.

* **`preview`** — содержит данные о карточке, выбранной пользователем для предпросмотра (тип `ICard`).

---

##### Методы

* **`constructor(events: IEvents)`** — в конструктор класса, передается объект для работы с событиями. Сохраняет переданный объект `events` в соответствующее свойство класса.

* **`setCards(cards: ICard[])`** — метод для сохранения массива карточек товаров. Принимает массив `ICard[]` и сохраняет его в свойство `cards`.


* **`getCards()`** — метод для получения массива всех карточек товаров.
    - **Возвращаемое значение:** `ICard[]` — текущий массив карточек товаров (`this.cards`).

* **`getCard(id: string)`** — метод для поиска и получения карточки товара по её уникальному идентификатору.
    - **Параметр:** `id` (`string`) — уникальный идентификатор искомой карточки товара.
    - **Возвращаемое значение:** `ICard | undefined` — найденная карточка товара либо `undefined`, если карточка с указанным `id` не найдена в массиве `cards`.

#### Класс `BuyerData`

interface IBuyerData {

    setAddress(address: string): void;
    setEmail(email: string): void;
    setPhone(phone: string): void;
    isOrderDataValid(): boolean;

##### Свойства

* **`events`** — объект для работы с событиями (тип `IEvents`), используется для оповещения других компонентов системы о произошедших действиях (например, о сохранении данных пользователя).
* **`payment`** — способ оплаты, выбранный пользователем (тип `TPayment`).
* **`address`** — адрес пользователя (тип `string`).
* **`email`** — электронная почта пользователя (тип `string`).
* **`phone`** — номер телефона пользователя (тип `string`).

* **`error`** — сообщение об ошибке (тип `string`), содержит описание проблемы при валидации данных (например, «Необходимо указать адрес»).

##### Методы

* **`constructor(events: IEvents)`** — конструктор класса, принимает объект для работы с событиями. Сохраняет переданный объект `events` в соответствующее свойство класса.

* **`getBuyerData()`** — возвращает частичные данные пользователя.
    * **Возвращаемое значение:** `Partial<IBuyer>` — объект с текущими значениями `payment`, `address`, `email` и `phone`.

* **`validateBuyer(buyerData: IBuyer)`** — проверяет корректность данных пользователя.
    * **Параметр:** `buyerData` (`IBuyer`) — данные пользователя для проверки.
    * **Возвращаемое значение:** `boolean` — `true`, если данные корректны.

* **`setPayment(method: TPayment)`** — устанавливает способ оплаты.
    * **Параметр:** `method` (`TPayment`) — выбранный способ оплаты.
    * **Действие:** сохраняет значение в свойство `payment`, генерирует событие `AppEvents.PaymentSaved`, выводит в консоль сообщение о сохранении метода оплаты.

* **`setAddress(address: string)`** — устанавливает адрес пользователя.
    * **Параметр:** `address` (`string`) — адрес для сохранения.
    * **Действие:** сохраняет значение в свойство `address`, генерирует событие `AppEvents.AddressSaved`, выводит в консоль сообщение о сохранении адреса.

* **`setEmail(email: string)`** — устанавливает электронную почту пользователя.
    * **Параметр:** `email` (`string`) — email для сохранения.
    * **Действие:** сохраняет значение в свойство `email`, генерирует событие `AppEvents.EmailSaved`.

* **`setPhone(phone: string)`** — устанавливает номер телефона пользователя.
    * **Параметр:** `phone` (`string`) — номер телефона для сохранения.
    * **Действие:** сохраняет значение в свойство `phone`, генерирует событие `AppEvents.PhoneSaved`.

* **`isOrderDataValid()`** — проверяет, достаточно ли данных для оформления заказа.
    * **Логика:** проверяет наличие значений в `payment` и `address`.
    * **Если данные валидны:** очищает свойство `error`, возвращает `true`.
    * **Если данные невалидны:** устанавливает в `error` сообщение «Необходимо указать адрес», возвращает `false`.
    * **Возвращаемое значение:** `boolean`.

* **`isContactsDataValid()`** — проверяет наличие контактных данных пользователя.
    * **Логика:** проверяет, заполнены ли поля `email` и `phone`.
    * **Если контакты валидны:** очищает `error`, возвращает `true`.
    * **Если контакты невалидны:** возвращает `false` (без установки сообщения об ошибке).
    * **Возвращаемое значение:** `boolean`.

* **`getError()`** — возвращает текущее сообщение об ошибке.
    * **Возвращаемое значение:** `string` — текст ошибки из свойства `error`.


* **`clearData()`** — очищает все сохранённые данные пользователя.
    * **Действие:** устанавливает значения `null` для свойств `payment`, `address`, `email`, `phone`.

#### Класс `BasketData`
Класс реализует интерфейс `IBasketData` и содержит данные о товарах, добавленных в корзину и методы для работы с ними.

```typescript
interface IBasketData {
    addCard (card: ICard): void;
    removeCard (id: string): void;
    getTotalPrice(cards: ICard[]): number;
    getCards (): ICard[];
    isInBasket (id: string): boolean;
}
```

### Свойства класса `BasketData`

* **`events`** — объект для работы с событиями (тип `IEvents`), используется для оповещения других компонентов системы об изменениях в корзине (например, о добавлении или удалении товара).
* **`cardsInBasket`** — массив товаров в корзине (тип `ICard[]`), хранит все карточки товаров, добавленные пользователем в корзину. Инициализируется пустым массивом при создании экземпляра класса.

---

### Методы класса `BasketData`

* **`constructor(events: IEvents)`** — конструктор класса, инициализирует экземпляр класса, принимая объект для работы с событиями. Сохраняет переданный объект `events` в соответствующее свойство класса и инициализирует массив `cardsInBasket` как пустой массив.

* **`addCard(card: ICard)`** — добавляет товар в корзину.
    * **Параметр:** `card` (`ICard`) — карточка товара, которую нужно добавить в корзину.
    * **Логика:** проверяет, есть ли товар с таким `id` в корзине с помощью метода `isInBasket()`. Если товара нет — добавляет его в массив `cardsInBasket` и генерирует событие `AppEvents.BasketChanged`. Если товар уже есть — выводит в консоль сообщение вида `${card.title} уже есть в корзине`.
    * **Действие:** обновляет состояние корзины и уведомляет систему об изменении.

* **`removeCard(id: string)`** — удаляет товар из корзины по его идентификатору.
    * **Параметр:** `id` (`string`) — уникальный идентификатор товара, который нужно удалить.
    * **Действие:** фильтрует массив `cardsInBasket`, исключая товар с указанным `id`, затем генерирует событие `AppEvents.BasketChanged`, уведомляя систему об изменении корзины.

* **`getTotalPrice()`** — рассчитывает общую стоимость всех товаров в корзине.
    * **Возвращаемое значение:** `number` — сумма цен всех товаров в массиве `cardsInBasket`.
    * **Логика:** использует метод `reduce()` для суммирования значений поля `price` у всех карточек в корзине.

* **`getCards()`** — возвращает массив всех товаров, находящихся в корзине.
    * **Возвращаемое значение:** `ICard[]` — текущий массив товаров `cardsInBasket`.

* **`isInBasket(id: string)`** — проверяет, находится ли товар с указанным идентификатором в корзине.
    * **Параметр:** `id` (`string`) — идентификатор товара для проверки.
    * **Возвращаемое значение:** `boolean` — `true`, если товар с таким `id` есть в корзине, `false` — если нет.
    * **Логика:** использует метод `some()` для поиска товара в массиве `cardsInBasket`.

* **`getCardsCount()`** — возвращает количество товаров в корзине.
    * **Возвращаемое значение:** `number` — число элементов в массиве `cardsInBasket` (общее количество карточек товаров).

* **`isEmptyBasket()`** — проверяет, пуста ли корзина.
    * **Возвращаемое значение:** `boolean` — `true`, если корзина пуста (`cardsInBasket.length === 0`), `false` — если в корзине есть товары.
    * **Логика:** сравнивает длину массива `cardsInBasket` с нулём.

* **`cleanBasket()`** — очищает корзину от всех товаров.
    * **Действие:** устанавливает массив `cardsInBasket` в пустое состояние (`[]`) и генерирует событие `AppEvents.BasketChanged`, уведомляя систему о том, что корзина очищена.
